Architecture

The Cloud Application

This is the cloud-based user interface you log into from your browser. It provides you visualization of cluster security information such as network traffic, vulnerabilities, Kubernetes network policies and the security policy and for taking actions such as switching the policy mode, adding policy rules, getting API tokens and exporting policy rules.

The Agent

The is installed on your Kubernetes cluster. It monitors traffic in the cluster, enforces your policy and communicates with the cloud application. 's capability of discovering new connections is referred to as Discovery.The agent contains the following services:

Kite

Kite communicates between the plugins and the application. It is installed on one node in the cluster

DNS Plugin

The DNS plugin identifies traffic in the cluster between services and to external services and blocks DNS requests that are not part of the security policy. It acts as a proxy, processing all DNS queries before passing them to the Kubernetes DNS pod.

Network Monitor

The network monitor discovers IP and UDP connection requests, including IP ingress. It works at the node level and employes the Conntrack daemon set.

Istio Plugin

Applies the Orca security policy to the cluster through Istio. Istio must be installed prior to installation.
