Architecture

The Cloud Application

This is the cloud-based user interface you log into from your browser. Used for visualizing cluster security information such as network traffic, vulnerabilities, Kubernetes network policies and the security policy and for taking actions such as switching the policy mode, adding policy rules, getting API tokens and exporting policy rules.

The Agent

The is installed on your Kubernetes cluster. It monitors traffic in the cluster, enforces your policy and communicates with the cloud application. The agent consists of the following primary services:

Kite

Communicates between the plugins and the application.

DNS plugin

  • Acts as a proxy, processing all DNS queries before passing them to the Kubernetes DNS pod.
  • Identifies traffic in the cluster between your services, as well as to external services.
  • Blocks DNS requests that are not part of the security policy.

IP Plugin

  • Employs the Conntrack daemon set.
  • Discovers IP connection requests, including ingress.
  • Discovers UDP connection requests.
  • Works at the node level.

Istio Plugin

Applies the Orca security policy to the cluster through Istio. Istio must be installed prior to installation.

Discovery (Discovery Engine)

The discovery engine is comprised of a subset of the plugins described above. It monitors your cluster and identifies the initiation of network connections. See Discovery Engine.
