Dashboard

The dashboard gives you a snapshot of your Kubernetes vulnerabilities through charts:

Network Policy Namespace Coverage

This chart compares the number of namespaces having at least one Kubernetes network policy with namespaces that have no network policies. The number of k8s network policies is not reflected here.

Namespace Vulnerabilities

This chart shows the number of namespaces for which vulnerabilities were found against the highest severity of vulnerability found. The number of vulnerabilities found in each namespace is not reflected here. The vulnerability level for a namespace is the largest vulnerability found in any of its containers.

Privileged Access

This is a list of containers and pods with privileged access. It comprises:

  • Containers for which privileged is set to true (privileged=true), therefore giving it access to all devices on the host.

  • Pods with a permissive pod security policy that will:

    • Let its containers share the host process ID namespace (HostPID=true) or
    • Let its containers share the host IPC namespace (HostIPC=true) or
    • Let the pod use the node network namespace (HostNetwork=true)

Kubernetes Vulnerabilities

This is a list of published vulnerabilities for your current k8s implementation, showing the CVE number and the version in which the vulnerability was fixed.
