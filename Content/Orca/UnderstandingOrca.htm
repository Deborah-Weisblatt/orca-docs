Architecture

The Application

The user interface for visualizing cluster security information such as network traffic, vulnerabilities, Kubernetes network policies and the security policy and for taking actions such as switching the policy mode, adding policy rules, getting API tokens and exporting policy rules.

The Agent

The monitors traffic in your cluster, enforces your policy and communicates with the Orca application. The agent consists of the following primary services:

Kite

Communicates between the plugins and the application.

DNS plugin

  • Acts as a proxy, processing all DNS queries before passing them to the Kubernetes DNS pod.
  • Identifies traffic in the cluster between your services, as well as to external services.
  • Blocks DNS requests that are not part of the security policy.

IP Plugin

  • Employs the Conntrack daemon set
  • Discovers IP connection requests, including ingress
  • Discovers UDP connection requests
  • Works for only - soon to be extended to all implementations
  • Works at the node level

Istio Plugin

Applies the Orca security policy to the cluster through Istio. Istio must be installed prior to installation.

Discovery Engine

Not a service in itself but rather a concept in that represents the identification mechanism of network traffic in your cluster. Comprised of other plugins. See Discovery Engine.
