Viewing and Copying Tokens

All currently defined tokens are listed in the API Tokens Window, which displays the following information:

Token Property Description
Scope

The intended use for the token:

  • VIEW - used for getting the information needed to display the badge in your git repository.
  • AGENT - used only by agents. Currently the Agent and SecureTrack integration.
  • ALL - used to make all other API requests to Orca. One example is for image scanning in build servers such as Jenkins. Contact support for more information.
Label A name describing the token. Free format text containing English characters, upper or lower case, spaces and hyphens
Token Value The value to be passed in the API call
Created Creation date and time

The token value can be copied to the clipboard by clicking the Copy to Clipboard icon on the appropriate line.

Learn how and why uses tokens and see what default tokens are provided for your convenience - Token Concepts.

Creating and Deleting Tokens

To create a new token:

  1. Click Create Token. The Add an API Token window is displayed.

  2. Click on the Scope line to display the available scopes and select the type required (described in Viewing and Copying Tokens):

  3. Enter a descriptive name in Label.
  4. Click Add Token. The token is added to the list.

To delete a token:

  • Hover over the line to be deleted
  • When the icon appears, click on it to delete the token.

Agent Installation Command

You can view the command line string needed for deploying on your cluster by clicking or Copy command line. Your deploy token is built into the command line. Once displayed, you can copy to clipboard by clicking the button that appears in the window. Click OK to close the window.

Kubernetes Network Policies

can prevent the Kubernetes network policies from blocking traffic and interfering with the learning process. This is controlled by the Set Kubernetes Network Policy to Allow-All in Learn Mode using the toggle switch

Enable or disable Set Kubernetes Network Policy to Allow-All in Learn Mode using the toggle switch. When set to enabled and Policy Mode is set to Learn, automatically creates an Any-Any-Allow Kubernetes network policy that allows all traffic, for each namespace. The purpose of this is to permit all traffic through the cluster and facilitate learning. An appropriate rule will be added for any new namespaces created while set to Learn. These -created rules will be removed if the appropriate namespace is removed or when Policy Mode is set to Enforce. No other Kubernetes network rules will be touched.
