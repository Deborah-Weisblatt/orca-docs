Policy Rules

Policy rules define the whitelist used by to determine what communication should be allowed between source resources and destination resources.

  • Add a policy rule
  • Remove a policy rule
  • Edit a policy rule

Add a Policy Rule

  1. Click to display the Add Rule Window.

  2. Enter fields. The fields that define a policy rule are:

    3. Field

    Description
    Scope Scope specifies whether traffic may originate from or be destined to servers outside of the application.
    • Application (default) - communication is allowed between a specific source asset and destination asset only within the same monitored application
    • Global - communication is allowed between a specific source asset and destination asset, regardless of which monitored application it is currently running in
    Source (Type) Source specifies a condition for receiving traffic from another location. First select the type.

    • Any (default) - permit traffic from any source. No additional parameters required.

    • Tags - permit only traffic from specified tags. When you select this option, you must insert one or more tags. As you type, a pop-up window is displayed containing the list of tags matching the characters you have entered so far. The list can contain entries of tag key only, tag key with tag value or a combination of both. If there are one or more matches on the key part of the tag, a single entry of the tag key will be included. If there are matches on the value part of the tag, the full tag value and key will be included. Click on the appropriate item from the list to add it to your selection. If the item you have chosen is a tag key without the value, the pop-up window will be replaced with another containing all tag values for the selected tag key. Click on the appropriate tag value to add to your selection. Repeat as necessary for additional tags. Once added, the tag can be removed from the selection by clicking on the icon.

      AND / OR: You can give multiple values for a tag key by separating values with a comma. These are considered 'OR' conditions.

      Example: Tier: DB, App, Applic (the rule applies to source assets that have a tag with keyword 'Tier' and values of 'DB' or 'App' or 'Applic'.

      If you add more than one tag keys, they are considered 'AND' conditions.

      Example Tier:DB Env:Prod (the rule applies to source assets having tags Tier:DB and alsoEnv:Prod)

    • Subnet - permit only traffic from a specified subnet mask. Enter a subnet mask in IPv4 format. Type a single mask followed by Enter. Repeat as necessary for additional subnet masks. Once added, the mask can be removed from the selection by clicking on the icon.
    Service (Type) Service specifies the network service permitted by the rule.
    • Any (default) - permit traffic using any service. No additional parameters required.
    • Network Service - permit only traffic using specified network services. As you type, a pop-up window is displayed containing the list of network services matching the characters you have entered so far (the match can appear anywhere within the network service name). Click on the appropriate item from the list to add it to your selection. Type or select a value of Any to include all services. Repeat as necessary for additional services. Once added, the service can be removed from the selection by clicking on the icon.
    • Cloud Action - permit only traffic using the specified cloud action. 2 actions are available - Storage: Read-Write and Storage Read-Only. As you type, a pop-up window is displayed containing the list of actions matching the characters you have entered so far (the match can appear anywhere within the network service name). Click on the appropriate item from the list to add it to your selection.

    Destination (Type)

    		 Destination specifies a condition for sending traffic to a remote location. Works the same way as source.
    Description A description of the policy rule

Delete a Policy Rule

  1. Click on the Ellipsis button for the rule you want to remove.
  2. Click Remove Rule from the pop-up menu.
  3. The rule will be removed from the display.

Edit a Policy Rule

  1. Click on the Ellipsis button for the rule you want to change.
  2. Click Edit Rule from the pop-up menu.
  3. The Edit Rule Window will be displayed. See Add a Policy Rule for more information.
  4. Make changes as necessary and click Save.

From the menu, click Policy.
