Assets

The screen lists the assets in your cloud accounts, their properties, accessibility and whether they are at risk. The list may be filtered when displayed, depending on how you got here.

The list of assets shows the following information:

  • Sequence of item in the list
  • Asset Type Icon - -Storage,-VM
  • The asset name, taken from the tag 'Name', instance id or the name of the object in the vendor account, depending on the type of asset
  • The application associated with the asset. Click on the application name to display the application.
  • The cloud vendor e.g. AWS, Azure
  • If the asset is assigned to a load balancer, the load balancer name is displayed
  • Warning If present, indicates a violation

  • Filter the displayed assets

  • View asset details

  • Resolve a violation

Filter the Displayed Assets

The list can be filtered on the following criteria:

  • Violations
  • Internet connectivity
  • Network peering
  • On-premises connectivity
  • Cloud vendor
  • Application name
  • Asset name
  • Load balancer name (the load balancer associated with the asset)
  • Tags
  • Reachable networks

Click on an asset in the list to display details of that asset.

View Asset Details

When clicking on an asset from the assets list or from other places in the system, the Asset Details Screen is displayed.

Select the required information by clicking on the Assets Menu:

  • Asset Overview (default)

    Properties of the asset including type, security entities, tags and VPC

  • Effective Access

    Inbound and outbound access considering both security entities and network connectivity

  • Security Access

    Access as defined by vendor security entities such as security groups and roles

  • Network Connectivity

    Access as specified by vendor routing configuration

  • Asset Policy

    Policy rules relevant to the current asset

Asset Overview

This is the default view when displaying the asset; or click Overview from the Asset Menu.

The information pertaining to the select asset is displayed in panels. At the top of the screen, basic information about the asset is shown followed by panels containing additional information.

Network - Gateway Access

The effective types of access possible to and from this asset: Highlighted if possible; Grayed-out if not possible:

  • Internet
  • Network peering
  • On-premises network

Security Entities

Lists all security settings associated with the asset e.g. security groups, roles and bucket inline policy.

Tags

Lists all tags that apply to this server, in the format key:value or just key. There is no differentiation here between asset-defined tags and custom tags defined in .

Server Properties Panel

The panel title is either Virtual Machine or Storage, depending on the server type. A number of server properties and their values are displayed.

For virtual machines the following properties are displayed:

  • Account
  • Avl. Zone
  • ID
  • IP
  • Region
  • Type
  • VPC
  • Vendor

For storage, the following properties are displayed: 

  • Account
  • Creation Date
  • Encryption
  • Region
  • Server Logging
  • Vendor
  • Versioning
  • Website

For more information see asset properties. For a more detailed explanation, see the cloud vendor documentation.

Effective Access

Effective access is access to and from the asset considering both security entities and network connectivity.

Click Effective Access from the Asset menu.

This screen has 2 tabs - Inbound and Outbound; select the appropriate tab. If no information is displayed, no access is allowed.

Effective Access Main Card

Each card in the list represents an end-to-end access route and includes:

Field Comments
+ Expand. Click to display the security access definitions and network connectivity that define the effective access. When clicked, changes to - symbol. Click again to collapse.
Source The point from which access can be initiated
Destination The point to which access is directed

Service

 The communications service that can be used by the access, can be multiple values, can include port number
Warning If present, indicates a violation
Options

(for violations only)

Click on the ellipses icon to view available options:

  • Investigate: Display the policy rules that are being violated in this effective access.
  • Add to Policy: See Resolve a violation

Click + to expand the card and display the security access definitions and network connectivity that collectively define the effective access; the + changes to -. Click again to collapse.

When the asset's communication is via a non-transparent load balancer, the two stages or hops are listed separately - the first from the source to the load balancer and the second from the load balancer to the asset.

Resolve a Violation

When there is violation, there are two ways it can be resolved:

  1. Add to policy - add a policy rule

    Enabled for effective access cards with a warning. Click on the ellipses icon on the effective access card and then click on Add to policy. The Add Rule window is displayed. Enter the appropriate details, explained in Policy.

  2. Delete from cloud - remove the cloud security rule from the VPC

    Enabled for security access cards with a warning, that have been expanded from an effective access card. Click on the ellipses icon and then click on Delete from cloud.
