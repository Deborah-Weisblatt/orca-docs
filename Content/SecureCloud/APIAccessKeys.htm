API Access Keys

API access keys are unique identifiers created in that allow other applications to connect to it. All keys are in the JSON web token format and once created, they can no longer be viewed in . You must therefore save the key immediately on creating it.

This function is available only to users with the Global Admin role.

The actions allowed and information provided through the API connection made will be dependent on the roles given to the API access key.

  • Name - the name given to the access key when it was created. It is unique and cannot be changed.
  • Role - roles that have been assigned to the key
  • Created - date and time the key was added
  • Add an access key
  • Edit an access key - hover over the key, click on and select Edit access key
  • Remove an access key - hover over the key, click on and select Remove access key

Add an Access Key

  1. Click on Add access key. The Add Access Key window appears.

  2. Enter a unique name for the access key.

  3. Assign a role:

    Example:

  4. Click Add. The access key details window appears. Do not close it or click on the screen outside of the window until you have saved both the access key name and the JSON web token to a safe location. Once the window is closed, will no longer be able retrieve the JSON web token, so if you have not saved it, the key will be useless.

  5. To save the access key name, hover over it and click on the copy icon , then paste to a safe location. This name will not be used in the API calls but you will need it if you need to identify the key in case other keys have been added.
  6. To save the JSON web token, hover over it and click on the copy icon , then paste to a safe location.
  7. Close the access key details window. The newly added key will appear in the list.
