Define your own risk criteria for your reports

  • Specify high risk ports in
  • Specify overly permissive network permissions
  • Specify the workloads to want to report as risks

Specify High Risk Ports in

Manage the list of ports you want reported as risks when exposed to the internet.

  • Enable / disable using the toggle

  • To add ports

    1. Click Add Ports
    2. Enter one or more ports in the format <[TCP][UDP]>: <port>, separating each port with a comma, e.g. TCP: 23, UDP: 67-68
    3. Click Add
  • Remove by clicking on the delete icon

Specify Overly Permissive Network Permissions

Specify the network permissions you consider as overly permissive. This is defined by two parameters:

  • Source subnet: Any, Larger than class A, Larger than class B, None
  • Services: Any, Any and TCP/UDP 0-65535, None

Enable / disable using the toggle.

To change values:

  1. Click on the edit icon
  2. Select the appropriate source and service
  3. Click Save

Specify Workload Security Risks

Specify the security contexts you want reported as risks.

A fixed list of pod and container security contexts is displayed. You cannot add additional security contexts.

Enable / disable using the toggle.

When enabled, mark/unmark the checkboxes as required.
