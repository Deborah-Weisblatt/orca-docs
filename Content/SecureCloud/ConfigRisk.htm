Specify in

What are ?. Manage the list of ports you want reported as risks when exposed to the internet.

Enable / disable this feature using the toggle. When disabled, no ports will be considered as risks when exposed to the internet and will not be included in the metrics and drilldowns or anywhere else.

To add ports:

  1. Click Add Ports
  2. Enter one or more ports in the format <[TCP][UDP]>: <port>, separating each port with a comma, e.g. TCP: 23, UDP: 67-68
  3. Click Add

Specify Overly Permissive Network Permissions

What is Permissive Access? Specify the network permissions you consider as overly permissive.

Enable / disable using the toggle. When disabled, no public cloud access will be considered as overly permissive when exposed to the internet and will not be included in the metrics and drilldowns or anywhere else.

To change values:

  1. Click on the edit icon
  2. Select the appropriate source and service
  3. Click Save

Specify Workload Security Risks

What are workload security risks? Specify the security contexts you want reported as risks. A fixed list of pod and container security contexts is displayed. You cannot add additional security contexts.

Enable / disable using the toggle. When disabled, no pod or container security contexts will be considered risks and will not be included in the metrics and drilldowns or anywhere else.

When enabled, check/uncheck the checkboxes as required. By default, all security contexts are checked.
