All exceptions you have created in and are listed.

Exception Name: As entered when creating the exception

Violation Type: Can be: Pod security context, Container security context or Container image vulnerability

Violation Details: The violations included in the exception, dependent on violation type.

  • For Pod security context - one or more pod security contexts
  • For Container security context - one or more container security contexts
  • For Container image vulnerability - a . Any that is better than the one displayed will always be included in the exception as well.

Resource Scope: The scope of resources (workloads) included in the exception. Can be:

  • The workload that caused the violation
  • All workloads having the specified container image
  • All workloads that contain the specified pod labels
  • All workloads

Cluster Scope: A single namespace, a single cluster or all clusters

Created: The user that created the exception with date and time created

Description: As entered when creating the exception

 

  • Delete an exception - hover over the exception then > Delete exception

    Once an exception is deleted, the affected assets or properties will again be marked as violating policy.

  • View affected assets ( exceptions only) - hover over the exception then > View affected assets

    A list of all assets affected by this exception is displayed.
