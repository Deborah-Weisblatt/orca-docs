Getting Started 1

How to get up and running fast with :

  1. Sign up.

  2. Perform initial setup.

  3. Read and understand the main concepts of global application visibility and policy.

  4. Monitor your public cloud security:

    • Make sure all your assets have asset tags or Custom Tags that identify your public cloud applications.
    • Tell how to identify your applications, using App Manager.
    • Get visibility and identify security risks by reviewing the Dashboard,
    • Configure Policy Rules to specify the communication that should be allowed between assets.
    • Discover violations and other features of such as , Application List and Assets.
    has a built-in onboarding 'wizard' that gets you through the first steps. It is popped up automatically the first time you log in to the system. Thereafter, you can run it whenever you like. Click on the Help icon at the top of the screen and select On-boarding from the menu.

  5. Monitor your Kubernetes clusters:

    • Review the cluster and its activity

      Use the following tools to get visibility into different levels of the cluster and expose vulnerabilities: Dashboard, Service Graph, Services, Nodes, Pods, Services, Nodes, New Connections, Kubernetes Network Policies.

    • Build and maintain a security policy. Review connections discovered by that you haven't allowed in the policy, add allowed connections to the policy and fine tune the policy by adding / removing as required.

    • Enforce the policy on Kubernetes. See Export as Network Policies.

  6. Add users to your account.

  7. Discover advanced Kubernetes features including automation and integration:

    • Set up immutable policy control - security policy creation built into your CI/CD pipeline.
    • Scanning of Docker container images for vulnerabilities.
    • Apply your policy on firewalls via integration with .
    • Apply your policy to Microsoft's Azure Firewall.
    • Put a badge on your repo to display the security score of your container.

  8. Use the reference guide as needed or, better still, spend an hour reading it all.
