Getting Started

Once you've understood the main concepts of global application visibility and policy, you need to decide how you want to identify your applications. You can use asset tags, server properties or a combination of both.

  1. Add the cloud accounts you want to monitor
  2. If you have decided to use tags to identify your applications, make sure they are defined consistently. If you want to use server properties, define them using Custom Tags.
  3. Tell how to identify your applications, using App Manager.
  4. Configure Policy Rules to specify the communication that should be allowed between assets.
  5. Review the Dashboard, ,, Application List and Assets screens to get visibility into the connections in your cloud accounts and to identify violations in policy.
