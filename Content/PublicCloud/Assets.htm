Assets

This page currently serves 2 purposes - a) all assets and b) assets for a selected application. It thus appears twice in the menu, each time with a different title. In the future, we'll break it up and turn it into 2 different topics using snippets

The screen lists the assets in your cloud accounts, their properties, accessibility and whether they are at risk. The list may be filtered when displayed, depending on how you got here.

The list of assets shows the following information:

  • Sequence of item in the list
  • Asset Type Icon - -Storage,-VM
  • The asset name, taken from the tag 'Name', instance id or the name of the object in the vendor account, depending on the type of asset
  • The application associated with the asset. Click on the application name to display the application.
  • The cloud vendor -
  • If the asset is assigned to a load balancer, the load balancer name is displayed
  • 'Internet exposed' if asset can be reached from the internet, considering both the security controls and the network connectivity.
  • Warning If present, indicates a violation

  • Filter the displayed assets

  • View asset details

  • Add exceptions so that certain violations are suppressed

  • View the [%=local-iris.ApplicationAssetMap%]/ Click on in the menu (only for assets shown for a selected )

Filter the Displayed Assets

The list can be filtered on the following criteria:

  • Internet exposure
  • Risk
    • Any risk
  • Cloud vendor
  • Application name
  • Asset name
  • Load balancer name (the load balancer associated with the asset)
  • Tags
  • Reachable networks

Click on an asset in the list to display details of that asset.

View Asset Details

When selecting an asset, the Asset Details Screen is displayed.

Select the required information by clicking on the Assets Menu:

  • Asset Overview (default)

    Properties of the asset including type, security entities, tags and VPC

  • Security Access

    Access as defined by vendor security entities such as security groups and roles.

  • Network Configuration

    Displayed for VMs only - the means by which the VM is defined to the network

  • Asset Policy

    Policy rules relevant to the current asset

Asset Overview

This is the default view when displaying the asset; or click Overview from the Asset Menu.

The information pertaining to the select asset is displayed in panels. At the top of the screen, basic information about the asset is shown followed by panels containing additional information. When hovering over fields, the copy-to-clipboard icon is displayed to allow easy copy/paste of screen information.

Exposure

Whether exposed to the internet. Two possible values:

  • Internet exposed
  • No internet

Security Control

Lists all the objects that control access to this asset: Security groups, roles, , bucket inline policies, VPC firewall rules and project IAM.

Properties

The properties shown vary from one vendor to another and are different for VMs and storage devices. See asset properties for details of each type of instance and see the appropriate cloud vendor documentation for further explanation.

Tags

Lists all tags that apply to this server, in the format key:value or just key. Includes both asset-defined tags and custom tags defined in .

Violations

All security issues found for the are listed according to the type of issue:

  • Risky Access
    • Risky ports
    • Permissive sources
    • Permissive services
  • Tag Usage - No tags

Security Access

Security access is the access to and from the asset defined by vendor security controls such as security groups and roles.

Select Security Access from the Asset Menu.

This screen can have 2 tabs - Inbound and Outbound; OutBound is shown only if relevant.

Each item in the list includes:

Field Comments
Effective Access Icon

Effective access is allowed or

Effective access is prevented via the network
Source The source asset IP, subnet or name or the security control applied to the asset from which access can be initiated. Can be Any.
Destination The destination asset IP, subnet or name or the security control applied to the asset being accessed. Can be Any.

Service

 The communications service that can be used by the access; can be multiple values, can include port number

Action

Allow / Disallow
Warning If present, indicates a violation
Options

Enabled only when warning icon shown above. Click on the options icon to view available options:

  • Compare to policy - displays all policy rules pertaining to the asset, below the security access. The same policy details are shown as when clicking Asset Policy.

  • Add to security policy (for policy violations only) - add a policy rule.

    The Add Rule window is displayed. Enter the appropriate details, explained in Policy.

  • Add exception (for and violations only) - add an exception that will no longer report this kind of access as a violation.

    The Add Exception window is displayed. Enter the appropriate details

  • Delete access from cloud - remove the cloud security rule from the VPC.

Add Exception

To add an exception:

  1. Select Security Access from the Asset Menu.

  2. Click the options icon on the access line that has the violation. The action menu appears.

    If the access has one or more violations due to or , Add exception will appear in the action menu and if the access has one or more , Add to policy will appear in the menu.

  3. Select Add exception from the options menu

  4. Complete the exception details.

    Exception Name: A name of your choice.

    Violation Type: Risky port, Permissive source, Permissive service. If more than one type of violation exists for the selected access then select the type of exception you are defining. If only one type of violation exists for the access, it cannot be changed.

    Violation Details: The ports, sources or services identified as violations. If more than one violation exists for the selected violation type and access, then select all those appropriate for the exception you are defining. If only one violation exists for this violation type and access, it cannot be changed.

    Asset Scope: The assets to be included in the exception. Select one: 

    • Security control - all assets to which access is permitted by the security control allowing this access
    • Asset tags - all assets having the exact same tags as this asset
    • Cloud account - all assets in the cloud account in which this asset is located
    • All cloud accounts - all cloud accounts monitored by SecureCloud

    You can click on View all assets in scope to view all assets that will be affected by this exception

    Description: Optional text of your choice.

  5. click Add to save.

Once added, exceptions can be viewed in , where they can be deleted if desired.
