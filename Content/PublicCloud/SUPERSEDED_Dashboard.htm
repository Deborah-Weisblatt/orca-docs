Dashboard

The dashboard is a high-level graphical summary of important security issues detected in your cloud account setup. It can be displayed for applications or assets and you can click on the View issue list link for a chart to display the applications or assets at risk.

  • Drill down to see the applications or assets at risk

  • Switch between the applications view and assets view
  • View the risk trends in your VPC in the charts below:

Risky Ports

Port 23, 110, 135-139, 445 and 3389 are considered high risk for various different reasons and allowing traffic to reach your assets from any of these could leave it open to attack. Click on View issue list to display all applications or assets allowing access through these ports.

Recommended best practice: In your cloud account security definitions, do not allow ingress from any of the ports listed above.

Permissive Access

The permissive access chart presents a comparison between applications or assets with ingress access (UDP or TCP) in any of the following circumstances. :

  • From any port (Min=0, Max=65535)
  • Via transport access from an unlimited range of IPs (class A subnet mask e.g. 255.0.0.0/8)
  • From any source (Source=ANY)

Click on View issue list to display all applications or assets that have permissive access. Recommended best practice: In your cloud account security definitions, avoid the use of unlimited ranges; allow traffic to assets only through well-defined ports and IP addresses.

Policy Violations

The policy violations chart presents a comparison between the number of applications or assets that are compliant with your policy and those that violate the policy. Click on View issue list to display all applications or assets with policy violations.

Recommended best practice: Review your applications and assets and resolve all violations.

Internet Access

The internet access chart presents a comparison between the number of applications or assets having access to or from the internet and those that don't have access. Click on View issue list to display all applications or assets that have access to the internet.

Recommended best practice: Limit internet access to as few devices as possible and ensure only approved assets have such access.

Tagging Alignment

The tagging alignment chart presents a comparison between applications or assets that have asset tags associated with them and those that do not. Click on View issue list to display all applications or assets that have no tags.

Recommended best practice: Use tags to define important asset characteristics such as application and environment, as described in Global Application Visibility.

From the menu, click Dashboard then click Applications Dashboard or Assets Dashboard.
