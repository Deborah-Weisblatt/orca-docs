The screen displays the cloud service provider accounts is currently monitoring, and their credentials. The information is separated into and .

Each line under Accounts represents one cloud vendor account and displays account information with an indication of the synchronization status for that account.

Each line under Credentials represents one set of user credentials that can assume a role to access one or more accounts under Accounts as an alternative to defining credentials at the account level.

You must define appropriate permissions in your cloud accounts to allow to monitor them. See AWS Permissions, Azure Permissions, Permissions.

Add an Account to

The required account must already exist on the cloud environment and have the required permissions.

  1. Make sure the Accounts Tab is selected

  2. Click Add Account. The Choose Vendor Window appears.

  3. Select AWS

  4. Enter Account Display Name. This is for your convenience only and can be any value you choose

  5. Select the connection type

    There are three different ways you can give access your AWS account:

    • Use the credentials of a user in the account
    • Give access to a user in another account via a role
    • Give access to 's own AWS account, via a role

    To specify the credentials of a user in the account:

    • Select Integration Method = Credentials
    • Enter Access key ID and Secret Access Key for the designated user in your AWS account

    To specify credentials of a user in another account:

    • Select Integration Method = Cross-Account Credentials
    • Enter the ARN Role (Amazon Resource Name) of the designated role in the target account that can be assumed by an authorized user from your other AWS account. Example: arn:aws:iam::006755140343:role/AssumRoleAdmin
    • Enter the external ID if specified in the role permissions for the account that will use the role
    • Select the Credential Name of the account that will use the role. This must be first saved - see Add a Cross-Account Credential.

    To specify Using 's own AWS account:

    • Select Integration Method = Use Tufin Account
    • Enter the desired ARN Role Name. Example: arn:aws:iam::006755140343:role/AssumRoleAdmin
    • Enter the external ID if specified in the role permissions for the account
  6. Click Add to save

Add a Account to

The required subscription must already exist on the cloud environment and have the required permissions.

  1. Make sure the Accounts Tab is selected

  2. Click Add Account. The Choose Vendor Window appears.

  3. Select Azure.

  4. Enter Account Display Name. This is for your convenience only and can be any value you choose

  5. Select the connection type

    There are two different ways you can specify the credentials needed by to access your account:

    • With single account credentials
    • With cross-account credentials, if you have previously saved cross-account credentials appropriate to the account to be added

    To specify a single account secret:

    • Select Integration Method = Credentials .
    • Enter your Azure Subscription ID.
    • Enter Tenant from your Azure active directory.
    • Enter Application ID from your application registration.
    • Enter Application Secret - a client secret from your application registration

    To specify a cross-account secret:

    • Select Integration Method = Cross-Account Credentials .
    • Enter your Azure Subscription ID.
    • Enter Tenant from your Azure active directory.
    • Select the desired Credential Name from the credentials you have saved previously - see Add a Cross-Account Credential.
  6. Click Add to save.

Remove an Account

  1. Click on options for the you want to remove.
  2. Click Remove Account from the pop-up menu.
  3. The Tag will be removed from the display.

Edit an Account

  1. Click on the options button for the you want to remove.
  2. Click Edit Account from the pop-up menu.
  3. The Edit Account Window will be displayed. See Add an Account for more information.
  4. Make changes as necessary and click Save.

Sync an Account

The sync function brings up-to-date cloud environment information for a specific account into .

  1. Click on options for the you want to sync.
  2. Click Sync account from the pop-up menu.
  3. The status tag icon on the account line will change to a syncing icon and once completed, the OK icon will be displayed. See Add an Account for more information.

Add a Cross-Account Credential

  1. Make sure the Credentials Tab is selected.

  2. Click Add Credentials. The Choose Vendor Window appears.

  3. Click on the vendor for which the account is to be added. The Add Credential Window is displayed.

  4. Enter Credentials Display Name. This is for convenience only and can be any value you choose.

  5. Enter the remaining fields according to your vendor:

    AWS

    • Enter Access key ID and Secret Access Key for the designated user in your AWS account

    Azure

    • Enter Application ID - the application ID of your Azure application registration
    • Enter Application Secret - a client secret from your application registration
  6. Click Add to save.

Remove a Cross-Account Credential

  1. Click options for the credential you want to remove.
  2. Click Remove credentials from the pop-up menu.
  3. The credentials entry will be removed.

Edit a Cross-Account Credential

  1. Click options for the credential you want to change.
  2. Click Edit credentials from the pop-up menu.
  3. The Edit Credential Window will be displayed. See Add a Credential for more information.
  4. Make changes as necessary and click Save.
