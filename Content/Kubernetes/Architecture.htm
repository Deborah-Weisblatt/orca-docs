Cluster Architecture

The Cloud Application

This is the cloud-based user interface you log into from your browser. It gives you a clear and easy to understand view of the cluster including workloads, network connectivity, vulnerabilities and Kubernetes network policies. Actions include switching the policy mode, adding policy rules, getting API tokens and exporting policy rules.

The

The is installed on your Kubernetes cluster. It monitors traffic in the cluster, discovers new connections, enforces your policy and communicates with the cloud application. services are deployed on your cluster in the Kube-System namespace and in a new namespace called Tufin-system, which is created in the installation process. DNS Plugin

The DNS plugin identifies traffic in the cluster between services and to external services and blocks DNS requests that are not part of the security policy. It acts as a proxy, processing all DNS queries before passing them to the Kubernetes DNS pod.

Network Monitor

The network monitor discovers IP and UDP connection requests, including IP ingress. It is deployed as a daemon set, on every node.

Istio Plugin

Applies the security policy to the cluster through Istio. Istio must be installed prior to installation.
