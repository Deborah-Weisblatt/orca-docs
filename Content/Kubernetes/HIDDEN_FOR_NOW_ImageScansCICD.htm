You can get to scan your container images for vulnerabilities automatically as part of your CI build process by adding some simple commands to the CI/CD script. For more information about docker image scanning see Image Scanning and .

Integrate the scan into the CI/CD pipeline

  1. Set environment variables:

    export TUFIN_DOMAIN=<>

    export TUFIN_PROJECT=<>

    export TUFIN_DOCKER_REPO_PASSWORD=<AGENT TOKEN>

    export TUFIN_API_KEY=<ALL TOKEN>

    Where,

    <> - Your name

    <> - The monitored

    <AGENT TOKEN> - in , Settings > API Tokens: Use the value of any token with Scope=agent and Label=kite

    <ALL TOKEN> - in , Settings > Settings > API Tokens: Use the value of any token with Scope=all

  2. Run the script:

    bash <(curl -s https://securecloud.tufin.io/api/scripts/image-scan)<IMAGE NAME>:<IMAGE_TAG>

    where :<IMAGE_TAG> is optional but also highly recommended. If not specified, the latest tag will be used, which doesn't reflect the version.

    3. then pushes the local image to the registry (registry.tufin.io), scans it and return the scans results.
