Services

The services window displays all the services in your Kubernetes cluster from nodes to which has access:

Column

Description
Name Name of the service
Namespace Namespace in which the service is running
Labels All labels defined for the service
Endpoints All endpoints exposed by the service that go through an ingress controller
Type

The type property in the service specification, which describes how the service is exposed:

  • ClusterIP (default): The service is exposed through an internal IP in the cluster, making it accessible only within the cluster
  • Nodeport: The service is exposed on each node's IP at a static port, making it accessible from outside the cluster
  • LoadBalancer: The service is exposed through a loadbalancer created in the current cloud with a fixed external IP assigned to it
  • ExternalName: The service is exposed via kube-dns as the name specified in the externalName property
Ports The service ports
Public IPs

External IP addresses of the service
Pod Labels Pod labels that are specified in all the pods making up the service
