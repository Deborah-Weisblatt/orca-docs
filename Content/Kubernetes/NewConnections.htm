New Connections

continually monitors traffic in your cluster, providing you with visibility into traffic flow and discovering new connections and in doing so, helps you formulate your security policy. Monitoring takes place whatever the Policy Mode.

What Type of Connections Does discover?

  • TCP and UDP connections
  • Connections between service pods
  • Connections from ingress to service pods
  • Connections from service pods to external end-points
  • Connections from the to pull your container images for analysis

Any traffic found that is not already listed as an allowed connection in your policy is automatically displayed in , where it will remain for up to one hour. Only one occurrence of a connection made between two points will be listed at any one time; if additional occurrences are discovered for a connection currently listed in the window, the time to keep it will be reset to another hour. stores the IP addresses for sources of inbound connections (ingress) and destinations of outbound connections (egress). These IPs are used when generating Kubernetes network policies.

The number of connections is displayed at the top of the screen. Each connection displays:

  • Source

    The name of the service from which the connection was initiated, or 'ingress' if the connection was initiated from outside the cluster

  • Source Namespace

    The namespace containing the source service

  • Destination

    The name of the service to which the connection was made or a domain/sub-domain name if the connection was made to a destination outside of the cluster

  • Destination Namespace

    The namespace containing the destination service

Connections - Allowed or Blocked?

When the Policy Mode is set to Learn, all new connections listed are allowed and when the Policy Mode is set to Enforce all new connections are blocked.

Notifications

A notification can be sent whenever a new connection is discovered. See Notifications and Alerts.

  • Turn new connections into policy rules (allowed connections)

    Select one or more new connections and click Add to Policy. The selected rules will be added to your security policy and removed from the current display and the number of new connections displayed will be updated accordingly.
