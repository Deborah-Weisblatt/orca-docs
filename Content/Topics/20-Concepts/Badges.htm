Overview

In case you aren't too familiar with badges, Github explains it nicely. "Badges are a unified way to present condensed pieces of information about your projects. They consist of a small image and additionally a URL that the image points to. Examples for badges can be the pipeline status, test coverage, or ways to contact the project maintainers."

Read more https://docs.gitlab.com/ee/user/project/badges.html

Badges are typically displayed in a code repository project's readme file to indicate the status of a process, but they can also be implemented elsewhere using an http request.

The Badge

gives away badges too! An badge shows the security score of a single container image (by default, the image last scanned). Clicking on the badge displays any vulnerabilities found for that image. You might want to insert the badges into the top of your code repository readme file something like this:

Image k8s.gcr.io/cluster-out64:

Image k8s.gcr.io/event-exporter:05:

Image calyco-vpcva-amd64:
