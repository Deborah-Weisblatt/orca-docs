Types of Policy

There are two different kinds of policy in - the policy and the Kubernetes policy.

The Main Menu Policy option is split into three distinct functions:

  • Building and maintaining the policy - add or remove allowed connections

  • Viewing Kubernetes network policies - see the pod-to-pod permissions currently defined in your Kubernetes policies

  • Viewing firewall tickets - see any currently open tickets from Tufin SecureChange

When clicking on Policy in the main menu, a sub-menu is displayed with options for each of these functions:

For building and maintaining the policy select Security Policy

For viewing the Kubernetes pod permissions select Kubernetes Network Policies

For viewing currently open firewall tickets select Firewall Tickets

Policy Mode

has 2 modes of operation or Policy Modes - Learn and Enforce. Learn mode lets you monitor your application without blocking traffic between services or containers so you can learn about the traffic connections between the services used in your Kubernetes cluster. Enforce applies your policy to the traffic in the cluster, allowing only traffic defined by the Orca policy rules. The monitoring and visibility capabilities such as Service Graph, Pods, and Services are always available, and are independent of the mode setting. Policy Mode can be changed in Settings.

  • Switch to Enforce when you are satisfied that the policy meets your enforcement criteria.

Kubernetes Network Policy and Learn Mode

can prevent the Kubernetes network policies from blocking traffic and interfering with the learning process. When this option is enabled (Settings) and Policy Mode is set to Learn, creates an Any-Any-Allow Kubernetes network policy (marked with a label), which allows all traffic, for each monitored namespace. These policies are removed if Policy Mode is set to Enforce or the option is disabled.
