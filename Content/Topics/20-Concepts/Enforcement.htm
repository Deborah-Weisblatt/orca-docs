Enforcement

How does enforce a security policy in your cluster?

enforces the security policy in your cluster using one or more mechanisms. Some are automatically applied when switching Policy Mode to Enforce, others must be initiated manually.

DNS

Enforced automatically when Policy Mode is set to Enforce.

At installation time, the DNS Plugin is installed in the Kubernetes DNS pod, which then acts as a proxy, processing all DNS queries before passing them to the Kubernetes DNS. When is set to Enforce, any DNS query that is not defined in as an allowed connection returns an NX domain (non-existent domain). This enforcement method is not active when is set to Learn.

Enforcement by DNS works for all supported Kubernetes implementations except , because it does not have a Kubernetes DNS.

Kubernetes Network Policies

Enforced manually. Export the security policy and import to Kubernetes network policies.

Apply on Firewalls

Enforced manually. You can implement the Orca security policy on through (requires SecureChange integration) or through Microsoft Azure Firewall. See Apply on Firewalls.

Istio

Applied automatically when Policy Mode is set to Enforce, when Istio is installed on the cluster. Integration with Istio is not included in the default installation. Contact Orca support for installation instructions.
