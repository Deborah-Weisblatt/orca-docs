Settings

Overview

The Settings window displays configuration information and lets you make changes such as creating API tokens, defining notifications and switching the policy mode.

Actions Available on this Screen

  • Manage API tokens
  • Get the deployment string needed to deploy the agent (Kite)
  • Switch between policy modes Learn and Enforce
  • Set up notifications
  • General actions

Manage API Tokens

API tokens are unique identifiers used by to give other applications access to it. will accept a request from another application only if the request is made with a token that matches that stored by . Some tokens are created automatically for the project ; others must be created manually.

Create a New Token

  1. Click Create Token. The Add an API Token window will be displayed.

  2. Click on the Scope line to display the available scopes and select the scope most appropriate for your requirement.

    • View for badge (in repositories such as GitHub)
    • Agent - for the Kite pod
    • All - for image scanning in build servers such as Jenkins
  3. Enter a label.
  4. Click Add Token. The token is added to the list.

Delete a Token

Hover over the line to be deleted; when the appears click on it to delete the token.

Copy a Token to the Clipboard

Click the Copy to Clipboard icon on the appropriate line to copy its contents to the clipboard

Default Tokens

Every project is provided with a number of tokens:

Name Description
badge Authentication token to use when integrating in your code repository, for displaying the Tufin badges with your Docker images
CI authentication token to use when integrating in your CI/CD pipeline
deploy authentication token to use when deploying in your cluster
kite authentication token use by the kite pod

Agent (Kite) Command Line

You can view the command line string needed for deploying on your cluster by clicking or Copy command line. Once displayed, you can copy to clipboard by clicking the button that appears in the window. Click OK to close the window.

Policy Mode

Click on the desired mode to switch.

Modifying Kubernetes Network Policies

You can extend the behavior you set in your security policy to your Kubernetes network policies. can automatically create and remove the appropriate Kubernetes network policy rules.

Set Kubernetes Network Policy to Allow-All in Learn Mode

When set to enabled and Policy Mode is set to Learn, a Kubernetes network rule will be created for each namespace that allows all traffic.

Auto-Generate Kubernetes Network Policies in Enforce Mode

When set to enabled and Policy Mode is set to Enforce, new Kubernetes network rules will be created to reflect the policy as defined in .
