Policy

The Policy option is split into three distinct functions:

  • Building and maintaining the policy
  • Viewing Kubernetes network policies
  • Viewing firewall tickets

When clicking on Policy in the main menu, a sub-menu is displayed with options for each of these functions:

Building and maintaining the policy Allowed Connections

Click on the appropriate option

 

A policy is a whitelist of traffic rules describing the communication that you allow between the services in your application. Policies are enforced by between services. Policy enforcement only occurs if learning mode is disabled. See Learning Mode Settings for configuration details.

 

 

 

What can I do?

  • Make changes to our policy
  • View Kubernetes network policies
  • View firewall tickets

Each of the options above can be selected from the sub-menu displayed when clicking on Policy.

Make Changes to Our Policy

Changes to policy are made on the Allowed Connections screen, which. lists rules that specify a permitted source namespace and destination domain name through which network traffic may pass. Initially the list is empty. Allowed connections must be added by the administrator.

Allowed Connections - What Can I do?

  • Add an allowed connection
  • Delete an allowed connection
  • Reset allowed connections
  • Filter - Filter by: namespace

Adding an Allowed Connection

  • Click on on the screen. The Add Allowed Connection window is displayed

     

  • Click on the empty namespace field under 'From' and select the appropriate namespace from the list from which to create the rule.

  • Click on the empty Domain Name field under 'to'. You can use a wildcard prefix * (e.g. *.example.com)

  • Click Add Rule.to save. The line will be added to the list.

Deleting Allowed Connection

  • Hover over the line for the desired connection and click when it appears at the end of the line.

  • The line will be removed.

Resetting Allowed Connections

  • Click on Actions at the top of the screen to display the actions menu.

  • Click on Reset Allowed Connections from the actions menu.

  • All allowed connections will be removed.

Kubernetes Network Policies

A read-only view of Kubernetes-defined network rules. The screen is split into two tabs, Ingress and Egress.

Under the Ingress tab, the polices are displayed in a numbered list and show the following information:

Field Description
# Display Sequence

Allow / disallow

Icon representing the type or status of the rule
Policy Name Kubernetes policy containing the rule
Target Namespace The Kubernetes cluster namespace to which traffic is destined
Pods The pods in the cluster namespace to which traffic is destined
Source Namespace The namespace outside the cluster from which traffic originates
Source Pods The pods in the namespace outside the cluster from which traffic originates
Source Subnet The subnet of the pods outside the cluster from which traffic originates

Port

The ports of the pods outside the cluster from which traffic originates

Under the Egress tab, the policies are displayed in a numbered list and show the following information:

Field Description
# Display Sequence

Allow/disallow

Icon representing the type or status of the rule
Policy Name Kubernetes policy containing the rule
Target Namespace The namespace in the Kubernetes cluster from which traffic originates
Target Pods The pods in the namespace from which traffic originates

Destination Namespace

The namespace outside the cluster to which traffic is destined

Destination Pods

The pods outside the cluster to which traffic is destined

Subnet

The subnets outside the cluster to which traffic is destined

Port

The ports outside the cluster to which traffic is destined

Firewall Tickets

A read-only list of tickets collected from all firewalls in the cluster. The screen is displayed below.

Field Description
Namespace Namespace in which the firewall resides
Endpoints Device on which firewall resides
Status Implementation status of ticket: Implemented, In progress, Failed
Message Message describing ticket status
Ticket

Ticket number
Last Update Last time ticket was updated

How do I get here?

  • From the menu in the left panel, click Policy and then choose one of the options displayed in the sub-menu:

    • View allowed connections
    • View Kubernetes network policies
    • View firewall tickets
