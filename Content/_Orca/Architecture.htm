Architecture

Currently Based only on k8s

The Cloud Application

This is the cloud-based user interface you log into from your browser. It provides you visualization of cluster security information such as network traffic, vulnerabilities, Kubernetes network policies and the security policy and for taking actions such as switching the policy mode, adding policy rules, getting API tokens and exporting policy rules.

The

The is installed on your Kubernetes cluster. It monitors traffic in the cluster, enforces your policy and communicates with the cloud application. 's capability of discovering new connections is referred to as Discovery.

services are deployed on your cluster in the Kube-System namespace and in a newly created namespace called Tufin-system. No services are placed in user or application namespaces.

At installation time, Kubernetes determines one node on which to deploy the Kite pod. Kubernetes deploys the Network Monitor daemonset on every node in the cluster, where permissions allow.

DNS Plugin

The DNS plugin identifies traffic in the cluster between services and to external services and blocks DNS requests that are not part of the security policy. It acts as a proxy, processing all DNS queries before passing them to the Kubernetes DNS pod.

Network Monitor (Previously Conntrack)

The network monitor discovers IP and UDP connection requests, including IP ingress. It is deployed as a daemon set, on every node.

Istio Plugin

Applies the security policy to the cluster through Istio. Istio must be installed prior to installation.
