Overview

API tokens are unique identifiers used by to give other applications access to it. will accept a request from another application only if the request is made with a token that matches that stored by . Some tokens are created for you automatically; others must be created manually. The list of defined tokens appears in Settings.

Tokens have three properties - scope, label and token value

Token Property Values
Scope

VIEW - used for getting the information needed to display the badge in your git repository

AGENT - used only by modules. Currently the and SecureTrack integration

ALL - used to make all other API requests to . Contact support for more information
Label Free format text containing English characters, upper or lower case, spaces and hyphens
Token Value The value to be passed in the API call

Default Tokens

Every project is provided with a number of tokens:

Scope / Label Description
view / badge Authentication token to use when integrating in your code repository, for displaying the Tufin badges with your Docker images
all / CI Authentication token to use when integrating in your CI/CD pipeline
view / deploy Authentication token to use when deploying in your cluster
agent / kite Authentication token used by the pod

Using API Tokens

The table below demonstrates how tokens can be used

Scope Format Example
VIEW https://orca.tufin.io/api/[VIEW PARAMETERS][TOKEN VALUE] See Implement Badges.
AGENT bash <(curl -s https://orca.tufin.io/api/[DOMAIN]/[PROJECT]/scripts/deploy-orca -H "Authorization: Bearer [TOKEN VALUE]")
