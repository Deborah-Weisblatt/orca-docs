App Manager

Specifying a Single Tag

When a single tag is specified in the App Manager, will identify an application for every different tag value for that tag key found on your assets. The total number of applications identified will be the number of different values in use for that tag on your assets, plus the default application if needed.

For example, a single tag key named App is specified in the App Manager and three different tag values for this key are found on your assets - hr, crm, ticketing, will identify that you have three different applications named hr, crm and ticketing.

Specifying Multiple Tags

When you specify more than one tag, identifies applications in two ways:

  1. One or more assets are found having any one of the tags specified - just like when a single tag is specified.
  2. One or more assets are found having more than one of the specified tags.

Application Naming Convention

  • Applications in are named automatically from the values of the tags specified in the App Manager. When a single tag is specified or one or more assets are found having any one of the tags specified, the application is named according to the value of one tag only. When multiple tags are specified, the application is named using all of the tag values found, separated by colon characters e.g. TagValue1:TagValue2:TagValue3.

    • Enter Tags

    1. Click on the icon.
    2. Enter the names of the tags that define applications.
    3. Click Save.
    Names of tags that have not been defined can be entered but will be ignored.
