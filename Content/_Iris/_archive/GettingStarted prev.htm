Let's Get Started

 

The Concept

uses the tags associated with your cloud assets to give you an application-centered view of your environment. All assets with the same value for the tags you specify are collected and displayed by as a single application. You can specify any number of both asset tags and custom tags to identify your applications. The custom tags are also displayed as part of the full collections of tags throughout . Policy rules define the whitelist used by to determine what communication is allowed between applications and between assets within an application.

To monitor your applications, requires:

  • Authentication information for the accounts to be monitored
  • A list of tags that should use to identify applications
  • (optional) Custom tags - Create custom tags based on asset properties. Custom tags can be used to augment the tags already defined for your assets, if your assets do not have a consistent set of tags that can be used to identify the applications you want to monitor.
  • A whitelist of policy rules that should use to identify the communication you permit between monitored applications and between services within applications.

has a built-in onboarding 'wizard' that gets you through the first steps. It is popped up automatically the first time you log in to the system.

What do I need to do?

  1. Add accounts: Go to Managing Monitored Accounts and add the cloud accounts you want to monitor.
  2. Configure how identifies applications: Go to Managing Applications and specify the tags that identify the applications.
  3. Create custom tags: (optional) If required, go to Managing Custom Tags and create custom tags.
  4. Configure policy rules: Go to Policy Rules and add the rules that identify what communication is allowed between applications and between assets within an application.
