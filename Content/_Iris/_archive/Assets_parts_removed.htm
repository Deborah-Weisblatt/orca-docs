Access Panel

This panel displays the access rules defined in the cloud vendor's security control mechanism, e.g. security group. The panel is split into two sub-panels - Inbound Access and Outbound Access. Switch between the two by clicking on the Inbound Access or Outbound Access tab.

The access paths to and from the asset are listed and contain the following information:

  • Source
  • Service
  • Destination
  • Action - allow or deny (deny is not supported by AWS)
  • Security Control
  • Warning icon
  • Ellipses icon - options

The warning icon indicates a potential risk in which a security group rule exists for the asset but there is no appropriate Iris policy rule.

When a conflict exists for inbound access, the ellipses icon is enabled indicating that there are options available to resolve it. See Add an Iris policy rule and Remove the Vendor Rule.

View the Security Group Rule for an Asset with no Appropriate Iris Policy Rule

Click the warning icon to display more information about the conflict.

The upper part of the displayed window shows details of the security group rule. If there is more than one warning, the number of warnings will be displayed on the window and you can page through them using the < and > symbols.

The lower part of the displayed window shows all the policy rules relevant for the asset.

Add an Policy Rule

Rectify a conflict by creating an policy rule - for inbound access only. Click the ellipsis icon and then click Add to Policy.

Remove the Vendor Rule

Rectify a conflict by deleting the rule from your cloud vendor's security group - for inbound access only. Click the ellipsis icon and then click Remove from vendor.

View the Application Associated With the Selected Asset

The name of the application that uses the selected asset is displayed at the top of the screen. Click on it to view the application.
